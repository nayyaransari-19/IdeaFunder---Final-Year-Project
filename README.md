# Idea Funder: University Project Incubation Platform

## 📌 Project Overview
Idea Funder is a full-stack, role-based platform built to bridge the gap between academic innovation and industry investment. The system provides a secure environment for students and faculty to showcase their projects, complete with an administrative verification pipeline. Once approved, verified investors can utilize a specialized dashboard to discover, review, and fund high-potential academic ventures.

## ⚙️ Core Modules & Role-Based Features

### 1. Administrator Portal
* **Project & User Management:** Complete oversight of the platform's user base and project lifecycles.
* **Verification Pipeline:** A dedicated workflow to review, approve, or reject submitted projects before they become publicly visible to investors.
* **Internal Communication:** Integrated messaging system to contact project creators directly for revisions or feedback.

### 2. Student & Faculty Portal (Creators)
* **Comprehensive Project Submission:** A secure upload flow requiring necessary documentation, technical details, and media.
* **Project Management:** Ability to manage, edit, and update active project submissions.
* **Account Control:** Full control over user profiles, credentials, and platform settings.

### 3. Investor Dashboard
* **Investment Management System:** A dedicated dashboard to track financial engagement and portfolio metrics.
* **Funding Analytics:** Differentiated views for projects funded individually versus group-funded initiatives.
* **Advanced Discovery & Search:** Ability to browse the entire marketplace of approved projects or filter specifically by keywords.
* **"Swipe-to-Discover" UI:** An interactive, swipe-based interface displaying quick-glance project summaries that seamlessly expand into comprehensive detail views.

## 🛠 System Architecture & Tech Stack
* **Frontend:** React.js (Component-driven UI, Swipe Mechanics)
* **Backend:** Laravel / PHP (RESTful APIs, MVC Architecture)
* **Database:** MySQL (Relational schema for Roles, Projects, and Funding)
* **Security:** Laravel Auth (Strict Role-Based Access Control)

## 📂 Repository Structure
**Backend (Laravel)**
* `app/` - Core logic, Eloquent Models, and Middleware
* `routes/` - API endpoint definitions
* `controllers/` - Business logic and request handling

**Frontend (React)**
* `components/` - Reusable UI elements
* `pages/` - Main application views and routing
